## 0. Prerequisites (create free accounts)
- [MongoDB Atlas](https://cloud.mongodb.com) вҖ” `MONGO_URI`
- [Cloudinary](https://cloudinary.com) вҖ” file uploads *(optional)*
- [Pusher](https://pusher.com) вҖ” real-time *(optional)*
- [Google Cloud Console](https://console.cloud.google.com) вҖ” OAuth + Drive *(optional)*
- A [GitHub](https://github.com) repo (done вң…) + [Vercel](https://vercel.com) + [Render](https://render.com) accounts

---

## 1. Database вҖ” MongoDB Atlas
1. Create a free **M0** cluster.
2. **Database Access** вҶ’ add a user + password.
3. **Network Access** вҶ’ allow `0.0.0.0/0` (fine for a demo).
4. **Connect вҶ’ Drivers** вҶ’ copy the SRV string вҶ’ this is your `MONGO_URI`.

---

## 2. Backend вҖ” Render
**Option A вҖ” Blueprint (recommended):**
1. Render dashboard вҶ’ **New + вҶ’ Blueprint** вҶ’ select this repo.
2. Render reads [`render.yaml`](render.yaml) and creates the `projectmanager-api` service.
3. Paste the secret env vars when prompted (at minimum `MONGO_URI`).
4. Deploy вҶ’ note the URL, e.g. `https://projectmanager-api.onrender.com`.

**Option B вҖ” Manual:**
1. **New + вҶ’ Web Service** вҶ’ connect repo вҶ’ **Root Directory:** `backend`.
2. Build: `npm install` В· Start: `npm start` В· Health check path: `/health`.
3. Add env vars (see checklist below) вҶ’ Deploy.

**Seed the database** (one-time): Render **Shell** tab вҶ’ `npm run seed`
(or run locally with the production `MONGO_URI`).

---

## 3. Frontend вҖ” Vercel
1. Vercel вҶ’ **Add New вҶ’ Project** вҶ’ import this repo.
2. **Root Directory:** `frontend` (Vite is auto-detected; [`vercel.json`](frontend/vercel.json) handles SPA routing).
3. **Environment Variables:**
   - `VITE_API_BASE_URL` = your Render backend URL (no trailing slash)
   - *(optional)* `VITE_PUSHER_KEY`, `VITE_PUSHER_CLUSTER`, `VITE_GOOGLE_CLIENT_ID`, `VITE_GOOGLE_API_KEY`
4. Deploy вҶ’ note the URL, e.g. `https://projectmanager.vercel.app`.

---

## 4. Wire them together (important!)
Back in **Render**, set/update:
- `FRONTEND_URL` = your Vercel URL
- `BACKEND_URL` = your Render URL
- `CORS_ORIGINS` = your Vercel URL

If using **Google OAuth**, add the callback to Google Cloud вҶ’ Credentials вҶ’ Authorized redirect URIs:
`https://<your-render-url>/api/auth/google/callback`

Redeploy the backend so the new env vars take effect.

---

## 5. Smoke test
1. Open the Vercel URL вҶ’ log in with the seeded demo admin (`admin@demo.com` / `Demo@12345`).
2. Create a project/task, upload a file, view the dashboard + dependency graph.
3. `GET https://<render-url>/health` should return `{ "status": "ok", "db": "connected" }`.

---

## вҡҷпёҸ Environment variable checklist

| Variable | Where | Required? |
|---|---|---|
| `MONGO_URI` | Render | вң… Yes |
| `JWT_SECRET` | Render | вң… Yes (auto-generated by blueprint) |
| `FRONTEND_URL`, `BACKEND_URL`, `CORS_ORIGINS` | Render | вң… after frontend deploy |
| `VITE_API_BASE_URL` | Vercel | вң… Yes |
| `CLOUDINARY_*` | Render | вҡӘ for file uploads |
| `PUSHER_*` / `VITE_PUSHER_*` | Render / Vercel | вҡӘ for real-time |
| `GOOGLE_*` / `VITE_GOOGLE_*` | Render / Vercel | вҡӘ for Google login/calendar |
| `EMAIL_USERS`, `EMAIL_PASSWORD` | Render | вҡӘ for email |

> Without the optional ones, those features degrade gracefully вҖ” the app still runs.

## вҸ° Note on Render's free tier (cold starts)
The free backend **sleeps after ~15 min idle**, so the first request after a nap takes
**30вҖ“60s**. Options:
- **Keep-alive ping** вҖ” a free uptime monitor (e.g. UptimeRobot / cron-job.org) hitting
  `/health` every 10вҖ“14 min keeps it warm.
- **Upgrade** Render to a paid instance (~$7/mo) for always-on.
- **Accept it** for a low-traffic portfolio demo (add a "first load may take ~30s" note).
